Sounds — how the event feed announces itself, on this device and on the host

🔊 Device sound

Play a cue in this browser on each new event — private to this device (stored in localStorage). Browsers block audio until you click, so the toggle itself is the unlocking gesture.

🎵 Custom event sounds

Assign an audio file (mp3, wav, ogg, m4a) to each event type. Files are stored only in this browser (IndexedDB) — nothing is uploaded or committed. An event type with no file uses the built-in synth cue. Playback respects the Device toggle above; Test plays regardless.

🖥️ Host cue

Sound on the host computer running the harness on each new event — shared by everyone and audible even with no browser open. It's event-determined: a start cue vs a finish cue (in voice mode, "agent … started" vs "… has finished"). Pick how live events sound, then audition each on the host right now.
