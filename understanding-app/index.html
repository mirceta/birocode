OpenSpec workflows — the jobs you'd actually do

OpenSpec is really two modes wearing one CLI. You READ the living truth — every read is a command, so it ports cleanly into the Console. You WRITE a change — but the thinking there is authored by hand; only the bookends (scaffold, gate, ship) are commands. Five workflows, and that's the whole job.

✓ runs in Console ✍ authored by hand ⌨ terminal only

How this ties to the app: the Read workflows are pure commands, so the Console tab runs them all. The Write change-loop is exactly where the 4-ported / 7-by-hand split on the Adopt tab comes from — its bookends (new change, validate --strict, archive) are buttons; its middle (delta specs, scenarios, design, tasks) you write yourself.
