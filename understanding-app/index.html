The two prose sections folded into the interactive map

The “How chat works” tab used to be one rich cytoscape map followed by two hand-built prose sections. They are now two overlays on the System-map view (“Show the binding” for §2, “⟳ Refresh!” for §3) — one visual grammar, the map as the single source. This page records why, and how. It worked out because the map already contained every node those sections talked about, so it was wiring, not modelling.

Done in native React — ChatMap.jsx / ChatGraph.jsx (overlay layer) + chatArchitectureData.js (bind/fate/twist annotations). No frozen snapshot to keep in sync.

The System map already holds these nodes

From data.js: the hero view is four nested where-it-runs boxes, and every piece §2 and §3 mention already lives in one of them. So folding in is mostly highlighting, not drawing.

conversation → run slot → session, traced through nodes that already exist.

Click any node — its role/desc panel already exists in the real map, so the §2 prose becomes a hover/click trace and the binding is just a colored path. Nothing new to model.

§3 is almost the map’s existing thesis

The map already says it: “everything in the device box is RAM/storage on YOUR device — the turn does not live here.” That is the refresh story. So §3 becomes a toggle overlay on the same diagram, not a separate cell grid.

wiped survives the twist
The one thing the spatial layout doesn’t already tell you: localStorage sits inside the device box yet survives (it holds the sessionId). That single exception is the part worth a glowing highlight — everything else the box boundaries explain for free.

Caveat that stays as text (a node footnote, not a box): the buffer is capped (~10k events, oldest trimmed). A very long turn can’t fully replay from after=0 — but loadTranscript already pulled the persisted part from disk, so only the live tail streams fresh.

Verdict

Fold both in. §3 → a red/green overlay + a stepped reconcile sim on the System map. §2 → a highlighted path. The nodes already exist, so it’s wiring, not modelling.

“Only the visualization” — mostly true

Structure reduces to the map. Two things still earn prose (best as the map’s click-to-read node panels):

  • the edge-case caveat (buffer cap ~10k) — a footnote, not a box;
  • a one-line orientation caption per view.

House rule: the app is a companion, not a replacement — so a thin prose layer stays. Pure-viz-only would cut against it.

The honest cost — how it actually landed

This was written assuming the live tab still iframed a frozen snapshot. It doesn’t any more — the tab became native React first, so the cost inverted in our favour:

  • work stayed in real React — ChatGraph.jsx gained a small data-driven overlay layer; ChatMap.jsx the toggle UI + reconcile steps;
  • §2/§3 became bind/fate/twist annotations on the existing chatArchitectureData.js nodes — highlighting, not new modelling;
  • no frozen snapshot, no SNAPSHOT.md sync — one source of truth;
  • still a normal build + redeploy cycle (it’s bundled JS), not a hot tweak.

Net: removed the bind-pills and the refresh-cell grid, leaving one visual grammar — the map is the single source of truth, and it’s edited as ordinary React, not a vendored snapshot.
