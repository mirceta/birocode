Local needs the first three. The public proxy adds rules 4 and 5 — the two traps that make a global deploy break in ways a local one never does. Source: docs/global-exposure-convention.md.

Why this work exists: the two sides are asymmetric. Local is taught by a product that genuinely practices its own contract. Global is taught from inside that same local product — so nothing in the repo actually demonstrates the public path.

🛰️ Local has a live example ✓

homepage/ is a real product that binds :5305 dual-stack and is served on the Local tab — it practices the 3-rule contract it teaches.

  • Live specimen: homepage/serve.mjs + assets
  • Hosts the "Local exposure" topic + copy-paste prompt
  • An agent reproduces it by doing what the page itself does

🌐 Global explainer only ✗

The "Global exposure" topic + prompt exist — but inside homepage/, a product on :5305 behind login. Nothing binds :5200 and crosses /preview/ for real.

  • Has: docs/global-exposure-convention.md, the homepage topic + prompt
  • Missing: a live product that practices the 5-rule contract
  • The real specimen (game-arcade) lives in a different repo

▼ the fix ▼

🚀 Build global-example/ — the public twin of homepage/

A tiny, build-less, self-contained product that binds 0.0.0.0:5200, is reachable through /preview/, and practices all five rules instead of only explaining them — hosting its own tutorial + paste-prompt, exactly like homepage/ does for local. See the Example app tab.

What makes global-example/ a faithful minimal example: it has a tiny stateful backend, so it doesn't just describe rules 4 & 5 — it fires them live. A "Bump" sends a body-ful POST, then a cache-busted GET.
