↳ The self repo is isSelf=true, pinned at index 0 — the default project. Today the Basic list is filtered by visibility only (Projects.jsx:127), so the self repo slips through and, being the default, its conversation renders even after you open another project.

Empty-state case — switch the scenario to “Only the self repo”: on a fresh harness where self is the only registered repo, hiding it means a Basic user has zero projects. That’s the correct outcome: better an honest empty state than the harness’s own conversation leaking to an End User.

why How the self conversation leaks — the resolution chain

Four steps from “pick a mode” to “render a conversation.” The leak happens because nothing in the chain subtracts isSelf for a Basic user. Green steps are where this change intervenes.

1

Mode is chosen on the device

Purely client-side, defaults to basic. The backend never learns the mode — it serves every repo (incl. self) to any authed client.

2

Repo list is filtered

Today: repos.filter(r => r.visibility === 'basic') — ignores isSelf. Fix: also exclude isSelf in Basic.

3

Active / default repo is resolved

Self is index-0 default, so a fresh or self-pointing Basic user lands on it. Fix: never resolve a Basic user's active repo to self — fall back to first Basic-visible repo (or empty state).

spec The three requirements (delta on project-visibility)

What the system SHALL guarantee once shipped. Each carries WHEN/THEN scenarios in the delta spec.

R1 The Self-Development repo is hidden from Basic-mode users

The Basic project selector excludes every isSelf repo regardless of its visibility. Advanced still sees it, pinned & non-removable.

R2 A Basic user's active repo is never the self repo

Even though self is the index-0 default / may be the persisted pick, resolution falls back to the first Basic-visible repo, else the empty state.

R3 The self conversation never renders for a Basic user

No dual harness chat, no self sessions — including immediately after switching projects, and after toggling Advanced→Basic while on the self repo.

scope Two decisions worth confirming

new capability

Seeded project-visibility

No existing spec owned repo visibility; plans/project-visibility.md was never folded into openspec/specs. Seed-and-grow per CLAUDE.md as we touch it, rather than bolting onto chat/access-control.

out of scope

Client-side enforcement only

UI mode is device-local; the backend can't know it. /api/repos & /api/sessions still return self to any client asking by id. A hard server-side gate needs the backend to learn the caller's mode — a separate, larger change.

Preserved invariant: self stays pinned at index 0 and non-removable for Advanced (RepoController.Remove still refuses IsSelf). This change only hides it from Basic.
