Wide tabs should fill, not float

Change tab-fill-available-width — a full‑span tab's content should stretch to fill its pane, instead of floating centered with empty side gutters.

Why it happens (and the fix)

1
The pane already stretches. In PaneStrip.jsx a spanned tab gets inline flexGrow: span, so the pane correctly grows to fill its tab‑spaces. That part works.
2
The page content caps & centers itself. Each page sets max-width + margin: 0 auto — e.g. .ck { max-width:1200px; margin:0 auto }. In a wide pane the content stops growing and floats in the middle. That's the gutter.
3
Fix: let content follow the pane's width. .app-content already declares container-type: inline-size, so each pane is its own size context. Drive the page width with @container queries: narrow pane keeps the comfortable reading cap; wide / full‑span pane fills 100%.
  • Touches CSS only: pages/cockpit.css, pages/settings.css, pages/terminal.css (+ maybe styles/global.css).
  • Untouched: span model & 1–4 range, tabWidths store, span controls, backend / API.
