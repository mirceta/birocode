Deploy to live, and the dead-man's switch

Shipping this repo's own Product (the Harness on :5099) is one committed, machine-independent script, swap.ps1. The risk it can't avoid by building carefully: a build that swaps in clean, passes its health check, then breaks down a few minutes later with nobody at the keyboard. The dead-man's switch covers exactly that - it auto-restores the last-good build unless you say "keep it". Run the simulation below.

guard never deploy a tree missing origin/main snapshot before the destructive swap arm 15-min auto-rollback keep it disarms

The committed pieces (all resolve paths from $PSScriptRoot)

deploy

swap.ps1

Guard → build → stage → snapshot → swap → restart → health. Then arms the switch, or rolls back if health failed.

arm

arm-rollback.ps1

Registers a one-time task to run rollback.ps1 in 15 min. Real DateTime trigger, never schtasks /SD.

recover

rollback.ps1

Restores run-bin.lastgood → run-bin via robocopy /MIR, restarts, then self-disarms.

disarm

keep.ps1

The "keep it" command. Deletes the scheduled task so the new build stays live.

    Two lessons ported verbatim (do not relearn them)

    2026-06-12

    Real DateTime, never schtasks /SD. /SD takes a locale-parsed date string; on this dd.MM.yyyy box it armed 06/12 as December 6th, so the rollback never fired and the harness stayed down. arm-rollback.ps1 uses New-ScheduledTaskTrigger -At <DateTime> - no string parsing.

    2026-06-12

    robocopy /MIR, not Remove+Copy. The mirror survives a held handle on the target dir itself; the old copy path once nested the build into bin\bin and the harness never came back up. Both swap.ps1 and rollback.ps1 mirror, and protect logs/ + appsettings.json with /XD + /XF.

    Verified 2026-06-21 on isolated temp dirs + a TEST-named task (17/17): parse/ASCII clean, snapshot+restore round-trip, mirror-away of stale files, runtime-state protection, the no-snapshot abort guard, and arm/disarm with a correct future trigger - all without touching live :5099.
