Understanding · a design decision, before we build it

Should the Console be organized by workflow?

The Console today is a flat palette — every OpenSpec command in one list. Fast if you already know what you want. But our reader is an expert in the old system, new to OpenSpec, and their hard question isn't what does validate do — it's when do I run it, and in what order. Flip the toggle to see the same commands answer that question.

Why this fits the rest of the app

The app already pairs a descriptive tab with an executable one. This change just completes the pattern — the Console becomes the executable shadow of the Workflows map, the same way it already shadows Adopt.

The honest cost

Verdict: worth it. Group the Console by workflow, keep it in Operate, keep the by-hand steps as muted context so the sequence reads. The redundancy with the Workflows tab is the good kind — read the job there, run the job here — as long as we keep that line crisp.
