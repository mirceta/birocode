Same layout everywhere. Both surfaces render the identical split from the shared FilesBrowser — no responsive drawer, no per-surface fork. The dock is just a narrow version of the same thing. Show/hide the folder browser (the ☑ toggle, top-right) collapses the whole left pane — tree + search — so the file view gets the full width; especially handy in the tight dock. The choice is remembered per device.

Where it renders

FilesBrowser.jsx

The shared core (the only file that grows): owns the tree, fuzzy search, open-file, pins, live-poll, and the new browserOpen collapse state behind useFeature('filesIdeMode').

pages/Files.jsx

Gate: new filesIdeMode, default 'advanced' — Basic keeps today's stacked tree-or-viewer. Toggle: show/hide folder browser, remembered in localStorage. Adjust: drag-resize the divider + A−/A+ tree zoom — both device-local prefs. Backend: none for MVP — reuses /api/files, /api/files/raw, pins.
