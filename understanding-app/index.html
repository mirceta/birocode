The Dashboard dock toolbar

openspec add-dashboard-dock-toolbar — curate which agent docks show in the grid, in place.

The problem. The Dashboard renders one tile per agent dock, and the fleet has grown to many docks. The only way to hide one from the grid was to leave for the Agents page and click its ▦ button. There was no in-place way to see the full roster and curate it.

The fix. A toolbar at the top of the Dashboard listing every dock as a tab — active means it renders in the grid, click to hide, click again to re-show. It drives the dock's existing dashboard field. No backend or schema change.

Try it

Click a tab to toggle whether that dock renders below. The toolbar always shows the full roster — hidden docks stay one click from returning.

The data flow

GET /api/dock → DockContext→ one roster, read two ways:
rosterTabs — unfiltered → the toolbar (all docks)
tabs = dockTabs.filter(d => d.dashboard !== false) → the grid
click a tab →updateTab(id, { dashboard: !active })→PATCH /api/dock/{id}

The grid re-renders from the changed tabs memo — no duplicate local state. Because the toolbar and grid read the one field, they stay consistent by construction.

Same field the Agents page uses

The Agents-page ▦ toggle calls the identical updateTab(id, { dashboard }). So a toolbar toggle is reflected there and vice-versa — one source of truth. Watch these cards track the simulator above:

Shared, not personal (the one real decision)

Reusing the synced dashboard field means hiding a dock hides it for every client and stays in lock-step with the Agents page — but it is not a private per-device view. The alternative was a device-local hidden-set (like claudeweb_dash_only_important). It was rejected as the default because it introduces a second, competing notion of "is this dock on the dashboard" that can silently disagree with the field and the Agents page. Shared reuse wins: zero backend work, one source of truth.

Gate: the toolbar lives inside the Dashboard overlay, which only renders under the agentDashboard (Advanced) gate — so Basic mode shows neither. No new capability entry.
