5 · The gap — now closed

Built and verified (OpenSpec change add-host-event-sound-rules, branch feat/host-event-sound-rules): the Sounds tab's "Event → sound rules" panel is real. Upload a wav or mp3 (max 2 MB) per event type — it's stored on the host, survives restarts, wins over Beep/Voice for that type, and the host plays it on matching events with no browser open. Each row has its own ▸ Test on host; types without a file keep the built-in cue, and a broken file falls back to it. Verified end-to-end on an isolated instance (API + headless-browser flow).

One skew to know about until it's deployed: the events-app page is served live from the working tree, so the new panel is already visible on live — but live's compiled backend doesn't have the rules endpoints yet, so uploads/tests there fail until the branch is merged and deployed. (Same UI-vs-binary skew that caused today's confusion.) Per-source rules remain future headroom.
