Prompt Plans

A plan is a named, ordered list of prompt steps you work through in sequence. Each step carries a name, a details body, and an expected result. They live next to your saved prompts — a new Plans tab in the chat composer's ⚙ pop-up — and like prompts they're global and backend-synced, so the library follows you across repos and devices. Clicking Use on a step drops it (plus its expected result) straight into the composer.

Decisions worth knowing

shape

Order is the sequence

Step order = send order. Reorder/edit/delete all just send the whole step array; the backend replaces the list wholesale. No per-step endpoints to drift out of sync.

scope

Global, like prompts

Not per-repo. The plan library is personal and follows you across every repo and device — the same choice PromptsService made.

durability

Atomic save, never reseed

Writes go temp-file-then-rename, so a kill mid-write can't truncate the store. An unreadable file is left untouched for forensics — defaults load in memory, your data is never clobbered.

limits

Bounded & cleaned

Name ≤ 80, step name ≤ 120, bodies ≤ 20k, ≤ 100 steps. Fully-empty steps are dropped so a stray blank row never persists. Name is the only required field — you can name a plan first, fill steps later.

nice-to-have

Paste-to-split

Paste a block using PROMPT: / DETAILS: / EXPECTED RESULT: markers and it carves into ordered steps. Forgiving — missing markers just leave fields empty.

verified

Browser-tested, then shipped

A Playwright run on an isolated build exercised create → add steps → reorder → Use → reload-persist, with no console errors. Then swapped onto live :5099 — health 200.
