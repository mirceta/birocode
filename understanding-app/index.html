Why the full-span tab finally fills the width

One tab, widened to span every visible slot, used to sit centered with gutters. Here's the actual reason — and the one decision that fixed it.

TL;DR. A tab widened to consume the whole pane budget left no room for neighbours, so the multi-pane layout decided “only one pane → not multi” and fell back to the classic single view, capped at 720px and centered. The fix: a deliberately-widened lone tab now renders as a single full-width pane in the uncapped strip instead of falling back.

1 · The decision that was wrong

Every render, PaneStrip.jsx → useMultiPane() packs as many tabs as the screen budget allows around the active one. It computes a window [lo, hi] of panes to show.

before

// only one pane fits -> bail to single view
if (lo === hi) return { multi: false }

// Layout then renders <Outlet/> inside
// .app-frame, which is capped:
.app-frame { max-width: 720px;
             margin: 0 auto; }  // gutters

A full-span tab fills the budget alone → lo === hi → single, capped, centered. The page-level CSS never even got a chance.

after

if (lo === hi) {
  // widened on purpose? keep it in the strip
  if (weight(active) > 1) return {
    multi: true,
    panes: [{ ...tabs[active],
             width: weight(active) }],
  }
  return { multi: false }  // truly alone
}

Now it renders as one pane in .app-frame--multi (no cap) → flex-grows to 100%.

3 · The complementary layer (page caps)

Once a tab is in the strip, a very wide pane could still center its own content if the page sets a reading-width cap (Settings caps at 560px; Cockpit/Terminal around 1100–1200px). So each pane's content area is its own size container, and the caps lift when the pane is wider than them:

/* the pane's .app-content is a container; the page reacts to ITS width */
@container (min-width: 560px) { .settings-page { max-width: none } }

This is why both pieces shipped: the layout fix gets the tab into the uncapped strip (the bug you saw), and the @container CSS stops page-level caps from re-centering inside an extra-wide pane.
