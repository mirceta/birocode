Prompt templates

A long custom prompt is mostly boilerplate with a few holes. Keep the template once; at use time, fill just the holes on a form — then the finished text drops into the composer.

1 · TemplateA custom prompt whose body carries {{name}} holes.
→
2 · DetectDistinct hole names, in first-appearance order, become the fields.
→
3 · Fill"Use" opens a form — one field per hole.
→
4 · InsertEvery {{hole}} is replaced; the result lands in the draft.

Try it — this is the real substitution logic

Holes you haven't filled stay {{like this}}; filled ones show highlighted. A template with no holes skips the form and inserts verbatim.

Why double braces

{{name}}A hole. Letters, digits, _, spaces and dashes; surrounding spaces ignored, so {{ ticket }} = {{ticket}}.
${VAR}Left alone — prompt bodies are full of shell/code, so single { } and ${ } must never be mistaken for holes.
{{file}}×3Repeated name = one field; filling it substitutes every occurrence.

The fields are derived from the body — never stored separately — so the holes and the declared fields can't drift apart. The backend record (Id, Emoji, Label, Text) is unchanged.
