How the OpenSpec flow writes its documents

A change moves Propose → Specify → Implement → Archive. The trick is knowing which file is written when. Click a phase (or press Play) — the matrix column, the file tree, and the notes all update to show exactly what gets created, edited, or moved.

openspec/ — what's touched now

The one rule that explains the whole diagram:
  • Everything in changes/<id>/ — proposal, design, tasks, delta spec — is written while the change is in flight (propose + implement).
  • The living baseline openspec/specs/<cap>/spec.md ("what the system does today") is written only at Archive, when the delta is folded in.
  • Archive does two moves at once: it folds the delta into the baseline, then relocates the whole change folder to changes/archive/<date>-<id>/ (the history).

Live example in this repo: add-operator-message-history sits at Phase 1 (Propose) right now — its proposal.md, tasks.md (0/10) and delta specs/operator-message-history/spec.md exist under openspec/changes/; nothing in openspec/specs/ has changed yet. It only will at Archive.
