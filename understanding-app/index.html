Why it works this way: if the new build crashed the harness, you couldn't use the harness to fix it — the phone UI would be dead. So the default action is revert, and keeping the build requires a positive signal.

The moving parts

swap.ps1
The one canonical deploy: origin/main guard, stage-before-stop, swap, health-check, then arms the switch.
run-bin.lastgood
Snapshot of the live build taken before the swap — the restore source.
ClaudeWebAutoRollback (scheduled task)
The timer. A real DateTime trigger (never schtasks /SD — locale bug). Currently set for 22:59.
rollback.ps1
Fires on the deadline: robocopy /MIR from last-good, restart live. Runs unattended.
keep.ps1
The disarm. Deletes the scheduled task. This is what "keep it" triggers.
