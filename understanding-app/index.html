Queued-prompt dock border

When an agent has prompts queued (waiting to send once it's free), its dock tile is painted with a thick black border — so in a grid of many agents you can see at a glance which ones have work lined up, without opening each.

Live dock — click a tile to queue / clear prompts

queued prompts (black, 6px) important (red, 6px) recency (blue, 5px) waiting (amber glow)

Click any tile to toggle a queued prompt on it. Notice the important and recency tiles: the moment they have a queued prompt, the black border wins — because a pending prompt is actionable and rarer. The amber waiting glow is a box-shadow, so it still layers around the black border.

How it's decided

tab.stash.length > 0The tile reads the agent's own queued-prompt list (the per-agent stash from /api/dock). Non-empty → add the --queued modifier.
6px solid #000Matches the dock's thick-border scale (the red --important border is also 6px), and black is reserved for this one state.
precedenceThe --queued CSS rule sits after important / recency / colour with equal specificity, so it overrides them while prompts are queued.

Pure presentation — one className modifier + one CSS rule, on all three tile renderers (dashboard phones, dashboard cards, and the Agents list). No backend or state change; the stash already drives re-renders.
