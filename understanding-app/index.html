Portable deploy — one swap.ps1 any agent can run

Deploying the Harness to live :5099 worked for one agent and was impossible for another — the classic machine-dependent trap. The deploy “mechanism” was a local, untracked script with hardcoded paths, and the safety chokepoint the docs promised didn’t exist in git. The fix: one committed, path-independent swap.ps1 with a real guard.

Why one machine could deploy and the other couldn’t

🖥️ Machine B (other agent) — before

✗ No deploy script in the checkout
✗ self-dev.md points to a swap.ps1 that doesn’t exist
✗ Assumes locked in-place bin\Debug
Can’t deploy — nothing committed to run

The five root causes — and the fix for each

untracked

Script lived in a gitignored dir

The working deploy was .selfdev-build/deploy.ps1 — never committed, so no other checkout had it.

→ Commit swap.ps1 at the repo root.
hardcoded

Absolute machine paths

Pinned to C:\Users\km\…; broke on any other user or folder.

→ Resolve everything from $PSScriptRoot.
fiction

Chokepoint that didn’t exist

self-dev.md promised a swap.ps1 enforcing the origin/main guard. No such file in git.

→ Make it real; rewrite the doc at its source (PreviewDoc.cs).
wrong target

Assumed the locked bin

Live runs from a copy (run-bin), not in-place bin\Debug (locked while serving).

→ Discover the live process; swap into a standard run-bin.
unsafe

Guard was never implemented

Even the local script skipped the origin/main check the doc claimed it enforced.

→ Real git merge-base --is-ancestor guard; abort = live untouched.
