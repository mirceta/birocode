Autopilot loop mode: re-send one fixed prompt until the agent is genuinely done

You’ve already told Claude everything it needs — but it still stops to ask “slice A or B?” and waits. Loop mode keeps re-sending your one prompt every time the agent finishes a turn, so it bulldozes through those questions, and stops the instant the job is genuinely done.

One prompt, resent automatically after every turn — until the agent signals it’s finished.

What it is
A deterministic loop. It does not classify or use any LLM “brain” — it resends one fixed prompt you define.
How it’s different
The shipped autopilot guesses which routine prompt fits (a classifier, brain still unbuilt). Loop mode is its simpler sibling — so it can ship now.
When the loop ends
A sentinel phrase in the agent’s reply (default LOOP_DONE), a risky-action deny-list hit, a hard iteration cap, or a run error.
