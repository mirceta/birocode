Discovery Eval — measuring "did we find every app?"

OpenSpec change add-discovery-eval: an offline harness that turns the quality of the dock's Discover Local Apps agent into a number, so the discovery prompt can be optimized with evidence instead of eyeballing.

The loop

1 · Golden fixturehard repo committed under tests/discovery-eval/fixtures/ + expected.json (the true apps, keyed folder+port)
→
2 · Real discoverythe SHIPPED path — LocalAppDiscoveryAsk → StructuredAskRunner → ClaudeMonitor, read-only tools — scan root = the fixture
→
3 · Scorerecall & precision on folder+port, explicit missing / extra lists
→
4 · Repeat ×Nagent is non-deterministic — report per-run scores, worst-case recall, runs-perfect count
→
5 · Prompt deltabaseline vs candidate prompts on the same fixture; pick the winner from the delta — adopting it is a separate change

Guardrails

  • Evaluates discover-local-apps; changes none of its behavior, endpoints, or dock UI.
  • Offline & dev-facing — needs the ClaudeMonitor gateway, so on-demand, not default CI.
  • Identity for matching is folder + port — names are agent-chosen and advisory.
  • Ground truth is committed and human-audited (expected.json + fixture README).
