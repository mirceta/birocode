The underscore bug 🪲

Why one repo's Claude conversations show as "no conversations" and a new chat is forgotten on refresh — even though the transcripts are sitting safely on disk. It comes down to one character in how a folder name is spelled.

2The two encoding rules, side by side

Same intent, different alphabets. The harness was written to handle only the three obvious path characters; the CLI sanitizes everything that isn't a letter or digit.

Harness — SessionService.EncodeCwd

workingDirectory
  .Replace(':','-')
  .Replace('\\','-')
  .Replace('/','-')
Replaces only : \ /. An _  (or ., space, …) survives untouched.

Claude CLI — what it really does

cwd.replace(/[^A-Za-z0-9]/g, '-')
Every non-alphanumeric character becomes -. Confirmed on disk: net8.0-windows → net8-0-windows, prg_worktrees → prg-worktrees.

3How that one character makes chats "vanish"

The data is never lost — the harness is just reading the wrong drawer.

Make the harness encode paths exactly the way the CLI does. Toggle to see the diff.
