The Local tab in Basic mode

Show the running product to the End User — without its authoring plumbing.

The Local tab embeds a project's running product through the harness's same-origin proxy. It used to be Advanced-only. This change makes it visible in Basic too — but view-only: an End User sees the product, never the add/remove/verify controls or the "how to embed" walkthrough. Flip the toggles to see exactly what renders, and which gate decides it.

One source of truth. Components never compare the mode directly — the capability map in UiModeContext.jsx holds localAppTab: 'basic', and the page reads const { isAdvanced: canAuthor } = useUiMode() to gate the authoring surface. Promoting the tab is a one-line map change; hiding the controls is one boolean. Self-repo safety is free: Basic users already can't select the Self-Development repo, so its local apps never reach Basic without extra code.
