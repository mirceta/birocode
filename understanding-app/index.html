Event Feed — Sound Design Playground

Audition candidate sounds for each event type. Everything here is synthesized live in your browser with Web Audio — no audio files — exactly how the real events-app "Device" cue works, so what you hear is what we'd ship.

🏛️ The wish: Age-of-Mythology-style unit acks.

Two ways to get there. The repo itself vendors no audio files (it stays build-less), so the built-in "AoM-spirit" theme below is an original, synthesized homage — snappy, playful RTS-selection cues with that "unit acknowledges your click" feel. But since this harness runs only for you at home, the real answer is bring your own sound: drop the actual game lines (or anything else) straight in — they live in your browser only, never in git.

The two sound layers today

🔈 Device per browser

Synthesized in the page watching the feed (your phone). Today: one triangle "blip" on every event. This is the layer this playground redesigns — cheap to give each event its own motif.

🖥️ Host shared

Plays on the machine running the harness (HostEventSound.cs): a beep, or a spoken "an agent has finished". Could also branch by type (e.g. say "started" vs "finished") — but it's shared & server-side.

Tip: hit ▸ start → end in the last column to hear the pair as a mini narrative (agent starts, then resolves). Design intent: start = rising/bright, end = settling/warm, error = low/dissonant, tool = a soft tick — so you can tell what happened without looking. The AoM-spirit row leans playful and snappy, like a unit answering a click.

Bring your own sound device-local · nothing uploaded

This is the real feature. Assign your own audio file to each event type. Since this runs only for you at home, drop in whatever you like — e.g. the actual Age of Mythology unit lines. Files are stored only in this browser (IndexedDB); nothing is uploaded or committed to the repo. An event type with no file falls back to the synth theme above. The shipped events-app uses this exact scheme (custom file wins, else built-in synth cue).

How this maps to the code

Each cell above is one playMotif(notes) call — a list of {freq, type, dur, gain} notes staggered in time, the same primitive the real blip() already uses. Shipping = replace the single blip() with a SOUNDS[eventType] lookup driving these motifs, keep the one Device toggle, and (optionally) pass the event type into HostEventSound.Notify() so the host cue can branch too. Pick the theme you like and it gets wired in via the OpenSpec change add-event-feed-sounds.
