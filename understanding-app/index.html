Activity / GitHub tabs

The events-app primary page split into two top-level tabs — with the "needs me" attention queue pinned above both, and display mode left flat.

Why. The page does two jobs: agent activity across the fleet (attention queue, per-machine sources, merged log) and GitHub administration (repo tiles, PR browser). They used to share one long scroll. Now each has its own tab — but the attention queue, the "needs me now" signal, stays above the tabs so an alert is never a tab-switch away.

How the active tab is chosen (on load)

PrioritySourceWhy
1?tab= in the URLA tab is linkable/bookmarkable — a shared link lands directly on it.
2localStorageDevice-local: reopening the page resumes the tab you last used, per device.
3default ActivityFirst visit with no signal lands on fleet activity.

Selecting a tab updates both localStorage and the URL (via history.replaceState) — no reload, and no extra board poll: one poll already paints both tabs, so switching is instant DOM show/hide.

Display mode is tabless

Display mode (?display=1) is the zero-interaction wallboard for the third monitor. A tab you can't click is pointless there, so the tab bar is hidden and both panels render together in one enlarged glance — attention + fleet + GitHub tiles — exactly as before this change. Any ?tab= value is ignored. Toggle it above to see the two panels flatten into one.
