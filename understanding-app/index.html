Both halves of "who am I to git" are now editable in the harness

OpenSpec change add-commit-identity-write is implemented, tested, and running live. The agent dock's git section always showed two identities — commits as (the repo's user.name/user.email) and pushes as (the GitHub account). Only pushes as was settable from the UI; the commits as half was read-only, so the only way to change a commit author was to drop into a terminal and run git config. This change closes that asymmetry: a new POST /api/git/identity writer plus an inline editor on the dock row. This app walks the split, the write path, the guards, and the first-ever C# tests.

Two identities, one used to be a dead end

Every commit records an author (git's user.name/user.email) and every push authenticates as a GitHub account. The dock has always shown both. The difference this change erases: the push side was already wired to a control (the dashboard PAT box sets the box's gh login), while the commit side was surfacing only — git-identity-surface was explicitly "read-only." Below: what each row is, where it comes from, and which was editable before vs after.

Local scope = the repo's own .git/config, a per-repo override. Global = the outer user config that every repo on the box inherits. This is the half the change makes editable.

pushes as unchanged

@mirceta-agents
source: GET /api/github-account — the box's authenticated GitHub login.

Already settable outside this change via the dashboard PAT control. This change does not touch it — the row stays read-only, and account switch/logout stay out of scope.

Key insight: these are independent. You can author commits as one identity and push them under a different GitHub account — a normal setup for agent commits. The change makes the author half controllable from the same place it's shown.

From row click to .git/config — the real code path

No new controller, no new service, no new module registration. The write rides the existing GitController, reusing the injected GitService, RepositoryResolver, and the run-busy guard (design decision D2). Every arrow below is an actual call in the shipped code.

DockIdentityRows.jsx edited client/src/…/DockIdentityRows.jsx

Edit affordance on the commits as row opens an inline name/email + scope form. On save: apiPost('/git/identity', { name, email, scope }, { repoId }), then calls onRefreshGit (the dock's existing git-status refresh, threaded down from PinnedAgent.jsx). Disabled while saving; inline error on failure. The pushes as row is untouched.

↓ POST /api/git/identity { name, email, scope }
GitController.SetIdentity added ClaudeWeb.App/Controllers/GitController.cs

Resolves the current repo. 409 if _runs.IsBusy(repo.Id). Maps scope ("global"→global, else local) and calls the writer. On success returns { ok:true, name, email, scope }; on the writer's failure returns 422 { ok:false, error }. Never throws to a 500 for a normal rejection.

↓ _git.SetCommitIdentity(repo.Path, name, email, scope)
GitService.SetCommitIdentity added ClaudeWeb.App/Services/Git/GitService.cs

Trims inputs; rejects when both empty (typed SetIdentityResult, no throw). Writes via RunGit(dir, "config --local user.name", value) / --global — values ride the ArgumentList, so no shell, no escaping. Partial writes allowed (only name, or only email). Then re-reads via ReadCommitIdentity and returns the authoritative { name, email, scope }. Any git failure degrades into Error, not a 500.

↓ git config --local/--global user.name/user.email
.git/config (local) or ~/.gitconfig (global) the repo's own config, or the outer user config

Local is the default and the safe choice for a per-repo dock. Global is offered but separately labelled because it changes every repo on the box (design decision D3).

↑ re-read → response → onRefreshGit → next git/status
Dock reflects the authoritative value

Four rules that keep the write safe

Busy guard · 409

Rejected while a chat run is active in the repo, consistent with merge/pull/push — so a commit identity can't change under an in-flight commit (D4).

if (_runs.IsBusy(repo.Id)) → Conflict

Empty rejected · 422

A request with neither name nor email mutates nothing and returns the empty-rejection result. No blank writes.

n=="" && e=="" → "Provide a name or email."

Partial writes · allowed

Git stores name and email independently, so setting only one is valid; the other keeps its previous value (D5). Inputs are trimmed.

only name → sets user.name, leaves user.email

No injection · literal args

Values pass through RunGit's ArgumentList — never a shell string — so a name or email needs no escaping and can't inject.

RunGit(dir, "config --local user.name", value)
Scope default is local (this repo only). Choosing global is deliberate and separately labelled — it rewrites the identity every repo on the box inherits.
