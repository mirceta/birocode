The first CSS attempt — .chat--composer-only { flex: 0 0 auto } — looked right and passed a glance. The browser tests showed the alternate views rendered at 0px. The culprit is how flexbox resolves an auto flex-basis: it falls back to the element's height property — and the chat's base rule sets height: calc(100dvh - …). Step through the chain, then flip the fix.
