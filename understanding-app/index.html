The underscore bug 🪲

Why one repo's Claude conversations show as "no conversations" and a new chat is forgotten on refresh — even though the transcripts are sitting safely on disk. It comes down to one character in how a folder name is spelled.

2The two encoding rules, side by side

Same intent, different alphabets. The harness was written to handle only the three obvious path characters; the CLI sanitizes everything that isn't a letter or digit.

Harness — SessionService.EncodeCwd

workingDirectory
  .Replace(':','-')
  .Replace('\\','-')
  .Replace('/','-')
Replaces only : \ /. An _  (or ., space, …) survives untouched.

Claude CLI — what it really does

cwd.replace(/[^A-Za-z0-9]/g, '-')
Every non-alphanumeric character becomes -. Confirmed on disk: net8.0-windows → net8-0-windows, prg_worktrees → prg-worktrees.

3How that one character makes chats "vanish"

The data is never lost — the harness is just reading the wrong drawer.

4Proposed fix one line

Make the harness encode paths exactly the way the CLI does. Toggle to see the diff.

Discover local apps · design 🛰️

  
Visualizing the OpenSpec change discover-local-apps: send a
  read-only agent to every registered repository to find apps that expose themselves as local apps,
  and get back a typed, validated list of { name, port }. Reproduces
  web-flow-autodev's structured-output pattern, reusing its
  ClaudeMonitor.Client as the send mechanism.

4The retry loop — bad JSON gets corrected, not accepted

  
A validating Parse rejects malformed findings; the runner feeds the bad reply +
    the error back and asks for a fix. Press play.

  

    
    
  

  






  
5Triggered from the agent dock — one click, one repo

  
Each agent dock in the dashboard is pinned to a repository. A "Discover local apps"
    button runs discovery for that dock's repo on demand and shows the structured JSON it returns.
    Click the button on a dock below.

  

  
Cross-repo coverage is per-dock, not a sweep: apps in another
    repository are found by clicking Discover on its dock — there is no automatic, all-repos fan-out.
    The prompt names no app and assumes no layout, so it works in any repo and as apps are added.






  
6Read-only by construction

  
The scan only reads. The reused client can't set plan mode, so read-only is enforced with a
    tool allowlist — every mutating tool is withheld.
