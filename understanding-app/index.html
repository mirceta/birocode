What the drawer is

  • One chronological list of every tool call in this conversation — not split across message bubbles.
  • Each row: ✓/✗/● status, tool name, input summary, and time.
  • Tap a row to expand its full input + output (same <details> style as the inline steps).
  • Each row is tagged by source: live (streaming now) or jsonl (rebuilt from the transcript).
Reuses the existing tool-event shape + ActivitySteps row look — no new visual language invented.

Why a drawer, not a route or new tab

  • It's a companion to the live chat, so it overlays in place — you keep your scroll position in the transcript.
  • It belongs to the active conversation only (v1) — no global dashboard, so it doesn't need its own page.

Two sources, merged by tool id — so the list never has holes

The drawer is fed by whichever source is available, deduped by tool id. Live wins while streaming; the transcript fills everything else.

live SSE
While a turn streams, ChatContext already folds tool events into steps. The drawer flattens those for the active conversation — rows appear in real time. → instant, but only for the running turn, only in this tab's memory.
GET /sessions/{id}/tools
On load / reattach, fetch the new endpoint. The backend walks the CLI JSONL transcript, pairs each tool_use with its tool_result by tool_use_id, and returns {id,name,summary,ok,preview,timestamp}. → durable, survives reload, covers turns started elsewhere.
merge
Dedupe by tool id → one complete, ordered list. A tool_use with no matching result shows as ● "no result / running". A malformed transcript line is skipped, never fatal.

The gap: tool calls vanish on reload

  • Live tool steps live only in client memory for the running turn.
  • On reload/reattach the transcript is rebuilt from GET /sessions/{id}/messages → SessionService.GetMessages, which deliberately strips tool_use/tool_result blocks.
  • So today there is no surface that answers "what has this agent actually done?" after a refresh.
The raw record still exists on disk in the CLI JSONL transcript — this change exposes it via a read-only endpoint. No new store is written; the transcript is the source of truth.

Deferred (out of scope for v1)

  • Cross-session / global tool-call dashboard or analytics.
  • Search / filter / export within the list.
  • Deep-linking a row back to its message bubble.
  • Showing thinking steps (those stay inline in the message stream).
