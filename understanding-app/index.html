Git Identity in Claude Web

Two independent identities, one global config — measured on this box (birocode), 2026-06-29.

There are two identities, and they are independent

Every git action touches one of two completely separate identities. Conflating them is the usual source of confusion.

Commit identity

Who a commit is authored as. Baked permanently into the commit object. Set by user.name / user.email.
user.name  = mirceta
user.email = kristijan.mirceta@gmail.com
source: ~/.gitconfig (global)
Used by: git commit. The harness runs git commit -m … with no identity argument, so it inherits this.

GitHub auth identity

Who you authenticate as when pushing or calling the GitHub API. A token, not baked into commits.
credential.helper = manager (GCM)
stored github.com token = none yet
gh CLI = installed, not logged in
Used by: git push, PRs, the GitHub API. The harness pushes with GIT_TERMINAL_PROMPT=0, so it relies entirely on this store.
They can disagree. A token for account X still produces commits authored as whatever user.email says. So a PAT textbox changes the auth identity — not the commit author.
Also: your commit email has a dot (kristijan.mirceta) while your Claude account has none (kristijanmirceta) — different addresses.

Global or per-repo? — Today it's global on both axes

Click a node to inspect it. Git supports per-repo overrides, but none are configured here.

~/.gitconfig · global identity
repo: birocode · inherits
repo: other-project · inherits
Git Credential Manager · auth store
host: github.com · one shared token
Select a node to see its config.
Because there is no includeIf in ~/.gitconfig and no local user.* in any repo, every repo commits as the same person. And because GCM keys credentials by host, every github.com repo shares one token. → One global place controls everything.

Where it surfaces — today vs proposed

The new account chips already answer "which GitHub account" — but globally, once, at the top. The dock git section shows no identity at all.

Dock git section — TODAY

agent · birocode
branch: feat/github-account-status+3 / -0 vs origin/mainclean
Branch + ahead/behind + actions. No "who".

Dock git section — PROPOSED

agent · birocode
branch: feat/github-account-status+3 / -0clean
NEWcommits asmirceta <kristijan.mirceta@gmail.com>
NEWpushes as not authenticated
Honest caveat: since identity is global today, these two rows would read the same in every dock. They're still worth showing — they're the effective identity for that repo, so the moment any repo gets a local override they'd correctly diverge.

Writes one host-keyed token (via GCM credential approve or gh auth login --with-token) that every github.com repo reuses. Sets auth, never the commit author. Stored secret → handled server-side, never echoed back.
