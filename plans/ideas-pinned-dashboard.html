On a narrow screen the left panel stacks above the grid (see open question 3).

Problem

While watching the agents on the dashboard, the user wants their Ideas (per-project notes) pinned to the left — always in view, not a separate tab you navigate away to.

Rough design (to refine)

  • Backend: none — reuse the shipped /api/notes (NotesController / NotesService, per-project via X-Repo-Id).
  • Frontend: split pages/Dashboard.jsx's .dash body into Ideas panel (left) + the existing .dash__grid (right). Extract the Ideas list/composer out of pages/Ideas.jsx into a shared component both the tab and the panel render (no duplication).
  • Gating: Advanced-only, like both parent features.
Load-bearing open question — whose ideas? The dashboard spans all projects, but ideas are per-project. The mockup shows the currently-selected project (label top-right of the panel). Alternatives: a project switcher in the panel, or a cross-project grouped list. Needs your call.

Open questions

  1. Whose ideas? Selected project (shown) vs. switcher vs. cross-project. ← decides the rest.
  2. Read-only or editable? Full composer/edit/delete, or a read-only glance.
  3. Mobile. Left panel collapses to a drawer / stacks on narrow screens (mockup stacks).
  4. Always shown or toggleable? Pinned vs. a show/hide toggle that frees grid width.
  5. Coordination. The agent dashboard is still evolving on feature/agent-dashboard; build on merged main and expect to rebase.
